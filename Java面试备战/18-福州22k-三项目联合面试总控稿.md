# 福州 22k 三项目联合面试总控稿

> 适用目标：福州 Java 后端 / 高级 Java 后端，期望薪资 `22k` 左右。
> 项目组合：`pms-substation-control` + `scheduledTask` + `rag项目`
> 用法：这份不是知识点堆砌，而是你后续模拟面试的总提纲。

---

## 一、结论先说

这三个项目要一起讲，而且要分主次：

- `pms-substation-control` 是主战场，证明你有真实复杂业务、微服务、消息、数据链路和稳定性经验
- `scheduledTask` 是能力拔高点，证明你不只是写业务接口，还能沉淀通用基础能力
- `rag项目` 是差异化标签，证明你能把 AI 能力落到业务系统里，不只是会调接口

最稳的讲法不是“三个平行项目”，而是：

> 我主线在变电 3.0 做生产管控和微服务治理，过程中沉淀了自研定时任务治理能力；在同一业务域里，我又把 AI 规则解析能力接进来了，做了自然语言到业务 DSL 的解析服务。

这样三块会变成一条成长线，而不是三段割裂经历。

---

## 二、面试权重怎么分

建议按这个比例讲：

- `60%` 讲 `pms-substation-control`
- `25%` 讲 `scheduledTask`
- `15%` 讲 `rag项目`

原因很简单：

- Java 岗首先看你是不是能扛住复杂后端项目
- 然后看你有没有抽象能力和中台能力
- 最后看你有没有差异化和业务延展性

如果把 RAG 讲太重，面试官容易怀疑你 Java 深度不够。
如果完全不讲 RAG，又浪费了你最有辨识度的亮点。

---

## 三、最推荐的自我介绍主线

你可以按下面这个版本去练：

> 我有 8 年后端经验，主线一直是 Java 后端。最近两年主要在电网生产管控场景做变电 3.0 微服务体系，核心工作包括多源数据同步、定时任务治理、RocketMQ 异步链路、Feign 失败补偿和告警工单相关优化。  
> 在这个过程中，我不只是做业务模块，还单独沉淀了一套定时任务治理 Starter，把任务注册、动态启停、执行监控、分布式锁、手动执行和异常通知做成了统一能力。  
> 另外我还在同一业务域里做了一个 AI 规则解析项目，把业务人员写的自然语言规则转成结构化 DSL，后面交给执行层做预警和工单生成。这个项目让我把知识库、混合检索、DSL 校验和回归验证真正落到了业务里。

这段话的作用：

- 第一句立住 Java 主线
- 第二句拉高工程能力
- 第三句制造差异化

## 四、三项目分别承担什么角色

### 4.1 `pms-substation-control` 你要证明什么

你要证明自己具备这些能力：

- 能做真实复杂业务，而不只是 CRUD
- 能处理多模块、多服务、多数据源协作
- 能做消息、缓存、限流、补偿、幂等等稳定性问题
- 能看懂并推进复杂链路，不怕脏活累活

建议重点讲这几块：

- 多模块 Maven 聚合工程，包含主服务、数据处理、事件调度、调度服务、公共包等
- 大量定时任务接入 `@MonitoredScheduledTask`
- RocketMQ 消费链路，尤其是故障跳闸消息的短时去重和失败重试
- Feign 调用失败后落库，再由后台任务重放补偿
- Redis 在这个项目里不只是缓存，还承担限流、去重、互斥

### 4.2 `scheduledTask` 你要证明什么

你要证明自己具备这些能力：

- 能从业务痛点里抽象出公共能力
- 能做注解 + AOP + Redis + 配置中心式的治理型组件
- 知道“能跑”和“可治理”是两回事

这个仓库里能拿来讲的证据很明确：

- `ScheduledTaskScanner` 启动后自动扫描并同步 `@MonitoredScheduledTask`
- `ScheduledTaskAspect` 统一处理启停判断、分布式锁、运行日志、调用计数、异常通知
- `ScheduledTaskLockManager` 用 Redis 锁 + 续租 + Lua 原子释放
- 支持手动执行绕过自动禁用判断
- 支持手动停止、应用关闭收敛、异常短信通知
- 状态不仅有成功失败，还有 `HAND_SHUTTING_DOWN`、`SYSTEM_SHUTTING_DOWN`、`REQUEST_LIMIT_EXCEEDED`

### 4.3 `rag项目` 你要证明什么

你不是去投 AI 算法岗，所以这里的重点不是“大模型多厉害”，而是：

- 你能把 AI 做成工程系统
- 你能把 AI 结果做成稳定、可回归、可校验、可上线的服务
- 你能跨技术栈，但主心骨还是后端工程能力

建议重点讲这几块：

- `PhaseAOrchestrator` 做整条编排，不是一个大 prompt 直接梭哈
- `RegexExtractor + AliasScanner + Disambiguator + LLMSlotAssembler` 的分层识别
- `BM25 + 向量检索 + RRF` 混合召回，`HybridRanker(k=60)`
- `DSLValidator` 和 `SlotValidator` 做生成前后双校验
- `SessionContext` 保存澄清状态、负例反馈、语料降权信息
- `backend/Makefile` 和 `backend/tests/` 说明它有完整回归链路，不是 demo

## 五、面试官最容易追问的主链路

### 5.1 先问变电 3.0

高概率问题：

- 你先介绍一下变电 3.0 项目
- 你负责的最核心模块是什么
- 为什么要做自研定时任务治理
- RocketMQ 怎么保证不重复消费
- Feign 调用失败你们怎么补偿
- Redis 在你们项目里除了缓存还做了什么
- 遇到过什么线上问题，怎么排查

### 5.2 然后追到 `scheduledTask`

高概率问题：

- 为什么不用 XXL-Job
- 你们这套任务平台具体解决了什么问题
- 动态启停怎么做
- 多实例怎么避免重复跑
- 手动执行和自动执行有什么区别
- 为什么要做调用次数限制
- 任务中断和应用关闭怎么收敛状态

### 5.3 最后问 `rag项目`

高概率问题：

- 这个项目和普通 ChatBot 有什么区别
- 为什么不是全交给 LLM
- 为什么要做混合检索
- 你怎么保证 DSL 合法
- 多轮澄清怎么做
- 怎么做回归验证
- 如果让我继续优化，你会先做什么

## 六、最稳的项目串联方式

不要这样讲：

> 我做了一个电网项目、一个定时任务项目、一个 RAG 项目。

要这样讲：

> 我的主线是变电 3.0。做这个项目时，我发现业务里有大量定时任务、任务治理和稳定性诉求，所以把这部分能力抽象成了单独的 Starter。再往后，业务里又出现了“规则配置复杂、依赖人工理解”的问题，所以我又做了规则解析引擎，把自然语言转成 DSL，接回原来的业务域。

这套讲法会让你显得：

- 做事有连续性
- 有从业务问题抽象平台能力的意识
- 有持续往上做的能力

## 七、哪些地方能深讲，哪些地方要收着讲

### 可以深讲

- `scheduledTask` 的任务扫描、切面治理、Redis 锁续租、状态收敛
- `FaultTripRecordConsumer` 的短时消息去重、重试上限、业务幂等分层
- `FeignFallbackAspect` 的失败落库字段设计
- `RateLimiterUtil` 的 Redis Lua 令牌桶
- `rag项目` 的编排链路、混合检索、DSL 校验、多轮澄清、回归体系

### 要谨慎讲

- 不要把 `pms-substation-control` 里所有能力都说成“你从 0 到 1 独立设计”
- 不要把 RAG 讲成自己是算法专家
- 不要把所有基础设施都说成是你一个人主导全栈完成
- 不要轻易报非常夸张的性能数字，除非你能解释压测口径

### 最佳口径

- 对主导过的能力，说“我主导设计和落地”
- 对深度参与的能力，说“我负责核心链路实现和优化”
- 对团队共建的能力，说“我重点负责其中哪一段”

这个档位通常会重点看四件事：

- 你是不是能独立扛模块
- 你是不是有线上问题处理经验

所以你的核心标签应该是：

- 复杂业务后端
- 稳定性治理
- 平台能力抽象
- 有 AI 落地经验但本职仍然是 Java 后端

## 九、最推荐的答题策略

每道题尽量按这个结构答：

1. 先说结论
2. 再说实现链路
3. 最后说效果、风险或取舍

比如“为什么自研定时任务平台”：

> 结论上，我们不是为了重复造轮子，而是业务更需要任务治理能力。  
> 具体上，我们把任务扫描注册、动态启停、执行日志、分布式锁、手动控制和异常通知统一收口到 Starter 里。  
> 这样做的收益是定时任务不再散落在各业务服务里，后续运维、排障和控制成本都低很多，但代价是我们要自己承担一部分平台维护成本。

## 十、你接下来怎么练最有效

建议按这个顺序练：

1. 先把 `变电 3.0` 的 30 秒、1 分钟、3 分钟版本说顺
2. 再把 `scheduledTask` 单独练成一个“为什么做、怎么做、解决了什么”的模块
3. 最后把 `rag项目` 压成“AI 工程化落地”的差异化补充
4. 等三块都顺了，再练串联版自我介绍

后续模拟面试的顺序也建议这样：

1. 第一轮只问项目介绍和亮点
2. 第二轮深挖 `变电 3.0 + scheduledTask`
3. 第三轮补 Java 八股、JVM、并发、集合、MySQL、Redis、MQ
4. 第四轮专练“压力追问”和“反问环节”

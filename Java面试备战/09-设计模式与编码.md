## 一、你项目中用到的设计模式

### 1.1 策略模式 + 工厂模式（Drools 规则引擎）

```java
// 不同预警类型用不同的规则脚本处理
// DroolsUtil 动态编译规则脚本，注入不同的 Service
KieSession session = DroolsUtil.getKieSession(ruleScript, orderService);
session.insert(orderData);
session.fireAllRules();
session.dispose();
```

**面试回答**：预警规则的判断逻辑会频繁变化，我们把规则和执行引擎解耦，规则用 Drools DRL 脚本定义，运行时动态编译加载，避免每次改规则都要重新部署。

### 1.2 模板方法模式（AOP 切面）

```java
// ScheduledTaskAspect 定义了任务执行的模板流程：
// 1. 前置处理（创建日志、设置上下文）
// 2. 执行目标方法（joinPoint.proceed()）← 子类/目标方法实现具体逻辑
// 3. 后置处理（更新日志、清理上下文）
```

### 1.3 观察者模式（Spring 事件）

```java
// ApplicationShutdownListener 监听 ContextClosedEvent
@Component
public class ApplicationShutdownListener
    implements ApplicationListener<ContextClosedEvent> {
    @Override
    public void onApplicationEvent(ContextClosedEvent event) {
        IS_SHUTTING_DOWN.set(true);
    }
}
```

### 1.4 代理模式（AOP + Feign）

- AOP 动态代理：切面增强原方法（监控、补偿）
- Feign 动态代理：接口注解生成 HTTP 调用实现

### 1.5 单例模式（Spring Bean）

- Spring 容器中的 Bean 默认 singleton
- `TaskExecutionRegistry` 静态 final ConcurrentHashMap 也是单例思想

### 1.6 责任链模式（RAG Pipeline）

```python
# 8 步 Pipeline，每一步处理后传给下一步
# 每步可以独立校验、降级、跳过
result = normalizer.normalize(input)
slots = slot_recognizer.extract(result)
validated = slot_validator.validate(slots)
# ...
```

### 1.7 建造者模式（DSL 构建）

```python
# DSL 逐步构建：
dsl = {
    "entity": slots.entity,
    "scenario": slots.scenario,
    "metrics": [...],
    "rules": [...],
    "output": {...}
}
# DSLNormalizer 进一步完善
# DSLValidator 最终校验
```

## 二、常见设计模式速查

### 创建型

| 模式 | 场景 | 一句话 |
|------|------|--------|
| 单例 | 全局唯一实例 | Spring Bean、配置类 |
| 工厂方法 | 创建不同类型对象 | 根据类型返回不同处理器 |
| 抽象工厂 | 创建产品族 | 不同数据库的 DAO 工厂 |
| 建造者 | 复杂对象分步构建 | StringBuilder、Lombok @Builder |

### 结构型

| 模式 | 场景 | 一句话 |
|------|------|--------|
| 代理 | 增强功能 | AOP、Feign |
| 适配器 | 接口转换 | 旧接口适配新接口 |
| 装饰器 | 动态增加功能 | InputStream 套娃 |
| 外观 | 统一入口 | MonitoredApiClient 封装 HTTP 调用 |

### 行为型

| 模式 | 场景 | 一句话 |
|------|------|--------|
| 策略 | 算法可替换 | 不同的限流算法、不同的排序方式 |
| 模板方法 | 固定流程，可变步骤 | AOP 切面的 around |
| 观察者 | 事件通知 | Spring Event、MQ |
| 责任链 | 请求依次处理 | Filter 链、Pipeline |
| 迭代器 | 遍历集合 | Iterator |
| 命令 | 请求封装为对象 | 定时任务的手动触发 |

## 三、SOLID 原则

| 原则 | 说明 | 你项目中的体现 |
|------|------|--------------|
| S 单一职责 | 一个类只做一件事 | Scanner 只扫描、Aspect 只监控、Configurer 只调度 |
| O 开闭原则 | 对扩展开放，对修改关闭 | Drools 规则脚本外部化，不改代码改规则 |
| L 里氏替换 | 子类可以替代父类 | Feign 接口的不同实现 |
| I 接口隔离 | 接口小而专 | 不同功能的 Feign Client 独立接口 |
| D 依赖倒转 | 依赖抽象不依赖具体 | Service 层依赖接口，不依赖具体实现 |

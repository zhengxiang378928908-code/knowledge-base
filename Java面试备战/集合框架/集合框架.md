# Java 集合框架

## 体系结构

```
Collection
├── List（有序，可重复）
│   ├── ArrayList   — 数组实现，查询 O(1)，增删 O(n)
│   ├── LinkedList  — 双向链表，增删 O(1)，查询 O(n)
│   └── Vector      — 线程安全的 ArrayList（已过时）
├── Set（无序，不可重复）
│   ├── HashSet     — HashMap 实现，O(1)
│   ├── LinkedHashSet — 保持插入顺序
│   └── TreeSet     — 红黑树，有序，O(log n)
└── Queue
    ├── LinkedList
    ├── PriorityQueue — 堆实现
    └── ArrayDeque   — 双端队列

Map（键值对）
├── HashMap        — 数组+链表+红黑树（Java 8+）
├── LinkedHashMap  — 保持插入/访问顺序
├── TreeMap        — 红黑树，key 有序
├── Hashtable      — 线程安全（已过时）
└── ConcurrentHashMap — 分段锁/CAS 线程安全
```

## HashMap 深度解析

### 底层结构（Java 8+）

- 数组 + 链表 + 红黑树
- 默认容量 16，负载因子 0.75
- 链表长度 >= 8 且数组长度 >= 64 时转红黑树
- 红黑树节点 <= 6 时退化为链表

### put 流程

1. 计算 key 的 hash → `(h = key.hashCode()) ^ (h >>> 16)`
2. 定位桶 → `(n - 1) & hash`
3. 桶为空 → 直接放入
4. 桶不为空 → 链表/红黑树查找
5. key 存在 → 覆盖 value
6. key 不存在 → 尾插法加入链表
7. 检查是否需要树化 / 扩容

### 扩容机制

- 容量翻倍（`newCap = oldCap << 1`）
- 重新 hash 分配：`hash & oldCap == 0` 留原位，否则移到 `原位 + oldCap`

### 为什么容量必须是 2 的幂？

`(n - 1) & hash` 等价于 `hash % n`，位运算更高效，且分布均匀。

## ConcurrentHashMap

### Java 7

- Segment 分段锁（默认 16 段）
- 每个 Segment 是一个 ReentrantLock + HashEntry 数组

### Java 8+

- 取消 Segment，改用 `Node 数组 + CAS + synchronized`
- 空桶用 CAS 插入
- 非空桶用 synchronized 锁住头节点
- 支持并发扩容

## ArrayList vs LinkedList

| 对比 | ArrayList | LinkedList |
|------|-----------|------------|
| 底层 | 动态数组 | 双向链表 |
| 随机访问 | O(1) | O(n) |
| 头部插入 | O(n) | O(1) |
| 尾部插入 | 均摊 O(1) | O(1) |
| 内存 | 连续，缓存友好 | 分散，额外指针开销 |

> **实际开发中 ArrayList 几乎总是更好**，因为 CPU 缓存局部性优势显著。

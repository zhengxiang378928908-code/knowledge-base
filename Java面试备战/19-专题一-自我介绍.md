# 专题一：自我介绍

> 目标：1 分钟讲完，让面试官记住三个标签：Java 后端主线 + 平台级治理能力 + AI 工程化落地。
> 练法：对着手机录一遍，听回放看有没有超时或漏点。

---

## 完整话术（直接背）

> 面试官您好，我叫郑响，8 年 Java 后端开发经验。
>
> 最近两年半我在做变电站生产管控平台，这是一个 7 个微服务模块的聚合工程，技术栈是 Spring Cloud Alibaba 加 RocketMQ、Redis、MySQL。我在里面的核心工作有四块：第一是多数据源的数据同步链路，要处理几千台设备、几十万条遥测记录的定时拉取和入库；第二是 RocketMQ 的异步告警链路，做了两层幂等、设备级熔断和消息配对；第三是 OpenFeign 的失败补偿机制，调用失败后自动落库，后台定时任务做健康检查再重放；第四是定时任务治理。
>
> 说到任务治理，这是我单独沉淀的一套 Starter，不是散落在业务代码里的几个 @Scheduled，而是一个完整的轻量级任务平台。它支持自动扫描注册、动态启停、分布式锁、执行监控、调用上限控制和启动恢复，目前管着 300 多个定时任务。
>
> 另外，我还在同一个业务域里做了一个 AI 规则解析项目。核心是把业务人员用自然语言写的预警规则转成结构化 DSL，用 Python 加 FastAPI 做独立服务，和 Java 主体系通过 API 交互。这个项目不是简单调一下大模型，而是做了多源 Slot 抽取、BM25 加向量混合检索、多轮校验和回译确认，有 68 条回归基线保证每次迭代不退化。
>
> 总结一下的话，我的主线是 Java 后端，核心能力在微服务治理和稳定性，同时有 AI 应用工程化落地的经验。

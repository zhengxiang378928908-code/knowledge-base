# 设计模式

## 六大原则

1. **单一职责**：一个类只做一件事
2. **开闭原则**：对扩展开放，对修改关闭
3. **里氏替换**：子类可以替换父类
4. **依赖倒转**：依赖抽象而非具体
5. **接口隔离**：接口精简，不强迫实现不需要的方法
6. **迪米特法则**：最少知道原则

## 创建型

### 单例模式

```java
// 枚举单例（推荐，天然防反射和序列化）
public enum Singleton {
    INSTANCE;
    public void doSomething() {}
}

// 双重检查锁
public class Singleton {
    private static volatile Singleton instance;
    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

### 工厂模式

- **简单工厂**：一个工厂类根据参数创建不同产品
- **工厂方法**：每个产品一个工厂类，符合开闭原则
- **抽象工厂**：创建一组相关产品（产品族）

### 建造者模式

```java
User user = User.builder()
    .name("Tom")
    .age(25)
    .email("tom@example.com")
    .build();
```

## 结构型

### 代理模式

- **静态代理**：编译期确定
- **JDK 动态代理**：基于接口（Spring AOP 默认）
- **CGLIB 代理**：基于继承

### 装饰器模式

动态给对象添加功能，IO 流的典型应用：

```java
new BufferedReader(new InputStreamReader(new FileInputStream("f.txt")));
```

### 适配器模式

将不兼容的接口转换为期望的接口（InputStreamReader 就是适配器）。

---

## 行为型

### 策略模式

定义一系列算法，让它们可以互相替换：

```java
// 支付策略
interface PaymentStrategy { void pay(int amount); }
class AlipayStrategy implements PaymentStrategy { ... }
class WechatStrategy implements PaymentStrategy { ... }
```

### 观察者模式

一对多的依赖关系，Spring 的 ApplicationEvent。

### 模板方法模式

父类定义算法骨架，子类实现具体步骤。AbstractQueuedSynchronizer 就是典型。

### 责任链模式

多个处理器形成链条，依次处理请求。Filter、Interceptor 都是责任链。

## Spring 中的设计模式

| 模式 | 应用 |
|------|------|
| 工厂 | BeanFactory |
| 单例 | Bean 默认 singleton |
| 代理 | AOP（JDK / CGLIB） |
| 模板方法 | JdbcTemplate, RestTemplate |
| 观察者 | ApplicationEvent |
| 适配器 | HandlerAdapter |
| 策略 | Resource 接口 |
| 责任链 | Filter / Interceptor |

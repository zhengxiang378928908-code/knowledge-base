# Java 并发编程

## 线程基础

### 创建线程的方式

1. 继承 `Thread`
2. 实现 `Runnable`
3. 实现 `Callable`（有返回值）
4. 线程池 `ExecutorService`

### 线程状态

```
NEW → RUNNABLE ↔ BLOCKED / WAITING / TIMED_WAITING → TERMINATED
```

### sleep vs wait

| 对比 | sleep | wait |
|------|-------|------|
| 所属类 | Thread | Object |
| 释放锁 | 不释放 | 释放 |
| 唤醒 | 超时自动恢复 | notify/notifyAll |
| 使用条件 | 任意位置 | synchronized 块内 |

## synchronized

- **修饰方法**：锁对象是 this（实例方法）或 Class（静态方法）
- **修饰代码块**：锁指定对象

### 锁升级（Java 6+）

```
无锁 → 偏向锁 → 轻量级锁（CAS 自旋）→ 重量级锁（Monitor）
```

## volatile

- **可见性**：修改后立即刷新主内存，其他线程立即可见
- **有序性**：禁止指令重排序（内存屏障）
- **不保证原子性**：`i++` 不是原子操作

### 经典应用：双重检查锁单例

```java
public class Singleton {
    private static volatile Singleton instance;

    public static Singleton getInstance() {
        if (instance == null) {
            synchronized (Singleton.class) {
                if (instance == null) {
                    instance = new Singleton();
                }
            }
        }
        return instance;
    }
}
```

## JUC 核心类

### ReentrantLock

```java
Lock lock = new ReentrantLock();
lock.lock();
try {
    // 临界区
} finally {
    lock.unlock();
}
```

vs synchronized：支持公平锁、可中断、超时、条件变量（Condition）

### AQS（AbstractQueuedSynchronizer）

JUC 的核心基础框架：

- 内部维护一个 `volatile int state` 和 FIFO 等待队列
- ReentrantLock、Semaphore、CountDownLatch 等都基于 AQS
- 独占模式（ReentrantLock）vs 共享模式（Semaphore）

### 常用同步工具

| 工具 | 用途 |
|------|------|
| CountDownLatch | 等待 N 个任务完成 |
| CyclicBarrier | N 个线程互相等待到达屏障点 |
| Semaphore | 限制并发数量 |
| Exchanger | 两个线程交换数据 |

## 线程池

### 核心参数

```java
new ThreadPoolExecutor(
    corePoolSize,    // 核心线程数
    maximumPoolSize, // 最大线程数
    keepAliveTime,   // 空闲线程存活时间
    unit,            // 时间单位
    workQueue,       // 任务队列
    threadFactory,   // 线程工厂
    handler          // 拒绝策略
);
```

### 执行流程

1. 核心线程未满 → 创建新线程
2. 核心线程满 → 放入队列
3. 队列满 → 创建非核心线程（不超过 max）
4. 全满 → 执行拒绝策略

### 拒绝策略

- `AbortPolicy`：抛异常（默认）
- `CallerRunsPolicy`：调用者线程执行
- `DiscardPolicy`：静默丢弃
- `DiscardOldestPolicy`：丢弃队首任务

### 为什么不推荐 Executors？

- `newFixedThreadPool`：无界队列，可能 OOM
- `newCachedThreadPool`：无限创建线程，可能 OOM
- `newSingleThreadExecutor`：无界队列，同上

> 推荐手动 `new ThreadPoolExecutor()` 明确参数。

## CAS 与原子类

- CAS = Compare And Swap（比较并交换）
- 底层依赖 `Unsafe.compareAndSwapInt`
- ABA 问题 → 用 `AtomicStampedReference`

```java
AtomicInteger count = new AtomicInteger(0);
count.incrementAndGet(); // 原子自增
```

## ThreadLocal

- 线程本地变量，每个线程独立副本
- 常用于存 userId、traceId、数据库连接
- **内存泄漏**：ThreadLocalMap 的 key 是弱引用，value 是强引用 → 用完必须 `remove()`

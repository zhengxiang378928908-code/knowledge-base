# Agent Guide

## 仓库定位

这是一个基于 Obsidian 的个人知识库，不是传统应用项目。仓库的核心目标是沉淀高质量、可复用、可快速检索的 Markdown 知识内容，当前重点包括：

- `Java面试备战/`：面试复习路线、专题笔记、速查卡片、资料沉淀
- `工具指南/`：终端、AI 工具、开发效率工具的使用说明
- `.obsidian/`：需要共享的最小配置

## Agent 的首要任务

当你在这个仓库中工作时，优先做下面几件事：

1. 帮助维护现有知识结构，而不是引入不必要的新结构
2. 优先把新增信息放进最合适的已有文档中
3. 在必须新增文档时，保持命名、层级和目录风格一致
4. 让内容更利于复习、搜索、复用，而不是只追求篇幅

## 编辑边界

- 不要把这个仓库当成代码工程去添加无关脚手架
- 不要大规模改写已有内容的语气和结论，除非用户明确要求
- 不要随意调整现有中文文件名、编号顺序和目录层级
- 不要提交 Obsidian 的本地工作区状态文件
- 修改知识内容时，尽量局部增量编辑，避免无意义重排

## 内容组织规则

### 1. 优先复用已有文档

新增内容前先判断是否已经存在可承载的位置：

- 同一主题的扩展，优先补充到原文档
- 速查型内容，优先放入对应主题下的 `知识点速查/`
- 原始资料、外部导出物，优先放入 `面试资料/`

只有在主题足够独立、内容较长、未来会持续增长时，才新增单独文档。

### 2. 延续现有命名风格

- 系列文档优先保持 `00-`、`01-`、`02-` 这种编号风格
- 文件名优先使用清晰中文语义，避免过度缩写
- 新建速查文档时，命名保持 `主题速查.md`

### 3. 保持 Markdown 结构清晰

- 每个文档只保留一个一级标题
- 优先使用二级、三级标题组织内容
- 适合对比的信息优先用表格
- 适合行动项的信息优先用编号列表或检查清单
- 命令、配置、示例优先用代码块

## 写作风格

- 语言以中文为主，表达直接、实用、可复述
- 优先写“面试怎么说”“工具怎么用”“为什么这样做”
- 少写空泛定义，多写场景、要点、对比、注意事项
- 尽量避免 AI 味很重的套话、重复总结和无效铺垫

## 知识库维护建议

当 Agent 被要求整理或新增内容时，推荐按以下顺序工作：

1. 先搜索仓库中是否已有相关主题
2. 判断是补充旧文档，还是创建新文档
3. 如果创建了新的系列或专题，补充对应的目录或导航文档
4. 检查是否有重复内容需要合并或互相引用
5. 对时效性强的内容标注整理时间

## 质量检查清单

提交前至少自查以下几点：

- 标题是否准确反映内容主题
- 内容是否能被快速浏览和定位重点
- 是否和已有文档产生重复
- 示例、命令、术语是否前后一致
- 是否保留了原有资料的核心信息和结构

## 当前仓库的默认假设

- 用户使用 Obsidian 作为主要阅读和编辑入口
- Markdown 是第一优先格式
- 这个仓库会持续迭代，不追求一次性写完
- 文档首先服务于个人复习、查阅和知识沉淀

如果没有额外说明，后续所有整理、补充、重构工作都应遵循本文件。
